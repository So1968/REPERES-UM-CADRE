
## Configuration conseillée du téléphone

Pendant la phase d’essai, le support principal est le téléphone de Michèle.

Les données saisies dans Pilotage UM sont stockées localement :
- sur le téléphone ;
- dans Chrome ;
- dans les données locales du site Pilotage UM.

Elles ne sont pas envoyées automatiquement à GitHub.

## Dossier de sauvegarde visible

Il est conseillé de créer un dossier dédié sur le téléphone :

Pilotage UM - Sauvegardes

Emplacement conseillé :

Mes fichiers
→ Stockage interne
→ Téléchargements
→ Pilotage UM - Sauvegardes

Ce dossier doit rester facilement accessible.

## Icône sur l’écran d’accueil

Il est conseillé d’avoir deux accès visibles :

1. Pilotage UM
   → raccourci de l’application web dans Chrome.

2. Pilotage UM - Sauvegardes
   → dossier contenant les exports JSON.

## Export JSON

L’export JSON est la sauvegarde de sécurité.

Routine conseillée :

- si des situations ou agents codés ont été ajoutés ou modifiés : faire un export JSON en fin de session ;
- si aucune donnée n’a été modifiée : pas d’export nécessaire ;
- avant tout changement important de version ou tout passage vers tablette : faire un export JSON.

## Téléphone plein

Si le téléphone manque fortement d’espace, le stockage local peut devenir moins fiable.

Risques possibles :
- export impossible ;
- téléchargement non créé ;
- Chrome ralenti ou instable ;
- risque de suppression accidentelle des données de site lors d’un nettoyage manuel.

Avant une utilisation régulière, il est recommandé de vérifier qu’il reste un minimum d’espace disponible.

## À ne pas faire

Ne pas effacer les données Chrome sans export JSON préalable.

Ne pas utiliser la navigation privée.

Ne pas supprimer le dossier Pilotage UM - Sauvegardes.

Ne pas envoyer les sauvegardes JSON par WhatsApp.

## Passage futur vers tablette

Si Pilotage UM est ensuite utilisé sur tablette, les données ne se synchroniseront pas automatiquement.

Procédure de transfert :

1. Sur le téléphone : Export JSON.
2. Sur la tablette : ouvrir Pilotage UM.
3. Ajouter Pilotage UM à l’écran d’accueil.
4. Import JSON.
5. Vérifier que les situations codées, agents codés et disponibilités sont bien récupérés.
