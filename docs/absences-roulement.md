# Absences, roulement et continuité — Repères UM 2.0

## Décision métier

Repères UM 2.0 reste un outil cadre personnel. La gestion des absences ne sert pas à surveiller les professionnelles. Elle sert à sécuriser la continuité du parcours UM, le roulement et les relais.

## Règle centrale

Une absence ne supprime pas la charge. Elle déplace temporairement la charge vers la continuité d’équipe.

La continuité prime sur le simple équilibre numérique.

## Types de disponibilité

- Présente
- Absence planifiée
- Arrêt maladie
- Absence imprévue
- Disponibilité réduite

## Arrêt maladie

Un arrêt maladie ne se projette pas comme une disponibilité future.

Même si une date de fin est renseignée, elle reste un point de vigilance. Elle ne vaut pas capacité projetée.

Tant que le retour effectif n’est pas confirmé, la professionnelle en arrêt maladie n’est pas proposée pour une nouvelle attribution.

## Absence planifiée

Un congé planifié ou une formation peut être pris en compte dans la projection si la date de retour est connue et si la nouvelle signature est postérieure au retour.

## Disponibilité réduite

Une disponibilité réduite reste dans le calcul, mais le score est ajusté selon le taux de disponibilité renseigné.

Exemple : score brut 10 avec disponibilité 50 % = score ajusté 20.

## Relais temporaire

Le relais est distinct de l’attribution.

- Attribution : nouvelle situation confiée à une professionnelle.
- Relais : situation déjà engagée, temporairement portée par une autre professionnelle.

Le relais temporaire peut être :
- vigilance ;
- actif ;
- complet temporaire.

Le relais augmente la charge de la professionnelle relais.

## Modification d’une fiche

Chaque fiche situation peut être modifiée. Cela permet :
- de changer la référente ou le binôme ;
- d’ajouter un relais temporaire ;
- de renseigner une prolongation ;
- de corriger un statut ;
- d’ajouter un historique cadre court ;
- de clôturer une situation.

## Limite de l’outil

Aucune donnée nominative ne doit être saisie :
nom, prénom, date de naissance, adresse, diagnostic, numéro DPI, détail familial ou donnée clinique identifiable.


## Affichage PC / téléphone

Le module disponibilité est affiché en cartes repliables. Sur PC, les cartes se placent en grille compacte. Sur téléphone et Samsung Galaxy Z Fold, une seule colonne est utilisée : seul le résumé est visible, les champs détaillés s’ouvrent au besoin avec le bouton Modifier/Replier.


## Affichage PC / téléphone

La disponibilité, les absences, le roulement et la charge équipe utilisent des cartes compactes repliables : le résumé utile est visible en premier, les détails ne sont ouverts qu'au besoin. Cette règle limite le bruit visuel sur téléphone, Z Fold et PC.


## Gestion de l’équipe active

L’équipe peut évoluer. Michèle peut ajouter un membre de l’équipe avec un code professionnel non nominatif, par exemple `IDE-04`, `EDU-07` ou `NEURO-03`.

Elle peut aussi retirer un membre de l’équipe active si la personne n’intervient plus dans le roulement. Les anciennes fiches conservent le code déjà renseigné afin de ne pas casser l’historique cadre.

Règle :
- ajouter / retirer concerne l’équipe active ;
- aucune donnée nominative n’est attendue ;
- le calcul d’attribution utilise uniquement les membres présents dans l’équipe active.
